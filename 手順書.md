# 刻印切り出し 完全セットアップ手順書(最終版)

## この仕組みでできること

Google Driveの「よちあい」フォルダに動画を入れるだけで、30分以内に自動で:
1. 刻印が最も正面で鮮明に読めるフレームを選定
2. 刻印部分を切り出して「容器番号_W重量.jpg」(例: 00001_W16.9.jpg)として保存
   ※読み取れなかったときは kokuin_0001.jpg の連番で保存(取りこぼしなし)
3. 「処理結果一覧.csv」に日時・動画名・画像名・読み取った文字を記録

鍵ファイル不要・カード登録不要・PC不要。設定は全部ブラウザで完結します。
監視フォルダと保存先フォルダはコードに設定済みです。

## パート1: GitHubのリポジトリを確認する

**重要: リポジトリが2つできている可能性があります。**
過去のエラー画面から、「-」(ハイフン1文字)という名前のリポジトリに
ファイルが入っていることが分かっています。

1. github.com にログイン → 右上の自分のアイコン → 「Your repositories」
2. 一覧を確認する
   - 「-」というリポジトリがある → **それを使います。開いてください**
   - 使うリポジトリを1つに決め、それ以外は混乱のもとなので
     Settings → 一番下の「Delete this repository」で削除推奨
3. 使うリポジトリの「Code」タブでファイル一覧を確認:
   - kokuin.py がある → パート2へ
   - 無い/古い → 「Add file」→「Upload files」で zipの中身のうち
     kokuin.py、requirements-light.txt、requirements-heavy.txt、README.md
     をドラッグして「Commit changes」

## パート2: 自動実行ファイルを作る(Actionsタブが無い人は必ずここ)

.github フォルダはドラッグで漏れやすいので、画面上で直接作ります。

1. リポジトリの「Add file」→「**Create new file**」
2. ファイル名欄に半角でこの通り入力: `.github/workflows/kokuin.yml`
   (スラッシュを打つと自動でフォルダになります)
3. 本文欄に、zip内の `.github/workflows/kokuin.yml` の中身を丸ごとコピペ
4. 「Commit changes」を2回
5. 上部タブに「**Actions**」が出現したことを確認(出なければ画面を再読み込み)

## パート3: Google側の設定(コピペ1回)

1. https://console.cloud.google.com にログイン
2. 画面上部のプロジェクトが「kokuin」か確認(無ければ新規作成して選択)
3. 右上の「>_」アイコンで **Cloud Shell** を開く
4. 下のブロックの1行目だけ書き換えて、全体を貼り付けてEnter:
   - リポジトリのURLが github.com/tanaka/- なら → "tanaka/-"

```bash
GITHUB_REPO="ユーザー名/リポジトリ名"   # ←ここだけ書き換え

PROJECT_ID=$(gcloud config get-value project)
PROJECT_NUMBER=$(gcloud projects describe $PROJECT_ID --format='value(projectNumber)')
gcloud services enable iam.googleapis.com iamcredentials.googleapis.com sts.googleapis.com drive.googleapis.com
gcloud iam service-accounts create kokuin-bot --display-name="kokuin bot" 2>/dev/null || true
gcloud iam workload-identity-pools create github-pool --location=global --display-name="GitHub" 2>/dev/null || true
gcloud iam workload-identity-pools providers create-oidc github-provider \
  --location=global --workload-identity-pool=github-pool \
  --issuer-uri="https://token.actions.githubusercontent.com" \
  --attribute-mapping="google.subject=assertion.sub,attribute.repository=assertion.repository" \
  --attribute-condition="assertion.repository=='${GITHUB_REPO}'"
gcloud iam service-accounts add-iam-policy-binding kokuin-bot@${PROJECT_ID}.iam.gserviceaccount.com \
  --role="roles/iam.workloadIdentityUser" \
  --member="principalSet://iam.googleapis.com/projects/${PROJECT_NUMBER}/locations/global/workloadIdentityPools/github-pool/attribute.repository/${GITHUB_REPO}"
echo "==== GitHubのVariablesに登録する値 ===="
echo "WIF_PROVIDER = projects/${PROJECT_NUMBER}/locations/global/workloadIdentityPools/github-pool/providers/github-provider"
echo "WIF_SERVICE_ACCOUNT = kokuin-bot@${PROJECT_ID}.iam.gserviceaccount.com"
```

5. 最後に表示された2つの値をメモ帳にコピー

## パート4: GitHubに2つの値を登録

1. リポジトリの「Settings」→「Secrets and variables」→「Actions」
2. 「**Variables**」タブ →「New repository variable」で2つ作成:
   - Name: `WIF_PROVIDER` / Value: projects/… で始まる長い方
   - Name: `WIF_SERVICE_ACCOUNT` / Value: kokuin-bot@… のメールアドレス

## パート5: フォルダにロボットを共有

1. Google Driveで「よちあい」フォルダを右クリック →「共有」
2. `kokuin-bot@…` のメールアドレスを貼り付け、「**編集者**」で追加
   (「Googleアカウントではありません」的な警告はそのまま進めてOK。
    通知の送信はオフでOK)
3. **保存先フォルダにも同じことをする**

## パート6: テスト実行

1. 「よちあい」フォルダにテスト動画を1本入れる
2. GitHubの「Actions」タブ →「刻印切り出し」→「Run workflow」→ 緑ボタン
3. 実行が一覧に出るのでクリックして待つ(動画ありなら5〜15分)
4. 緑✓ → 保存先フォルダに画像とCSVができていれば完成!
   以後は30分おきに全自動です

## 困ったとき(実際に起きたことベース)

**「Actions」タブが無い**
→ パート2の kokuin.yml がまだ無い。パート2をやる。

**「Get started with GitHub Actions」等のセットアップ画面が出る**
→ 同上。ワークフローのファイルが無い状態。パート2をやる。

**Streamlitの「ModuleNotFoundError」画面が出る**
→ 見る場所が違います。Streamlitは今回無関係の別サービス。
  そのタブは閉じて、share.streamlit.io でアプリを削除。
  確認はGitHubのActionsタブで。

**Driveで「動画の処理を続けています」と出る**
→ Google Drive自身の再生用変換の表示で、この仕組みとは無関係。
  異常ではありません。

**実行が赤×になる**
→ 実行をクリック → 赤い行をクリックするとエラー文が出る。
  - 「Googleに認証」で失敗 → パート3・4のやり直し。特にGITHUB_REPOの
    書き換え間違い(大文字小文字も一致させる)が多い
  - 「新着動画: 0件」→ パート5の共有漏れ
  - それ以外 → エラー文をコピーして相談

**緑✓なのに画像ができない**
→ ログに「新着動画: 0件」のはず。パート5の共有を確認。
  一度処理した動画は二度と処理されません(processed.jsonに記録)。
  同じ動画で再テストしたいときは、リポジトリのprocessed.jsonを
  開いて該当行を削除(またはファイルごと削除)してCommit。
