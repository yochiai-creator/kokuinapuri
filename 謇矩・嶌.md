# 刻印切り出し(GitHub Actions版)設定手順書

すべてブラウザだけで設定できます。カード登録・PCへのインストールは不要です。
所要時間はゆっくりやって30〜40分ほど。上から順に進めてください。

---

## パート1:Googleの「ロボット用アカウント」を作る(約10分)

プログラムがGoogle Driveに出入りするための専用アカウント
(サービスアカウント)を作ります。

1. https://console.cloud.google.com を開き、Googleアカウントでログイン
2. 画面上部のプロジェクト名をクリック →「新しいプロジェクト」→
   名前は何でもOK(例: kokuin)→「作成」
3. 画面上部の検索窓に「Drive API」と入力 →「Google Drive API」を開く →
   「有効にする」を押す
4. 左上の三本線メニュー →「IAMと管理」→「サービスアカウント」
5. 「+ サービスアカウントを作成」→ 名前は何でもOK(例: kokuin-bot)→
   「作成して続行」→ ロールは選ばず「完了」
6. 一覧に出てきたアカウントの**メールアドレス**
   (○○@○○.iam.gserviceaccount.com)をメモ帳にコピーしておく
7. そのアカウントをクリック → 上の「キー」タブ →「鍵を追加」→
   「新しい鍵を作成」→「JSON」→「作成」
   → **ファイルがダウンロードされる。これが後で使う「鍵ファイル」**

## パート2:Driveのフォルダに「ロボット」を入れる(約3分)

1. Google Driveで**動画をアップするフォルダ**を開く
2. フォルダ名の横の「▼」または右クリック →「共有」
3. パート1-6でメモしたメールアドレスを貼り付け、権限を「**編集者**」にして送信
   ※「通知を送信」のチェックは外してOK
   ※「組織外と共有できません」と出た場合はWorkspace管理者に相談が必要です
4. **保存先フォルダ**にも同じことをする
5. ついでに、2つのフォルダそれぞれをブラウザで開いたときの
   **アドレスバーのURL**をメモ帳にコピーしておく(パート4で使う)

## パート3:GitHubにアップロードする(約10分)

1. https://github.com でアカウントを作る(無料。メールアドレスだけでOK)
2. 右上の「+」→「New repository」
   - Repository name: kokuin(何でもOK)
   - **Private** を選択(重要:非公開にする)
   - 「Create repository」
3. お渡しした zip をPCで展開(右クリック→すべて展開)
4. GitHubの画面で「uploading an existing file」というリンクをクリック
   (見当たらなければ「Add file」→「Upload files」)
5. 展開してできたフォルダの**中身を全部**まとめてドラッグ&ドロップ
   ※「.github」というフォルダも含めて必ず入れてください。
     アップロード後のファイル一覧に
     「.github/workflows/kokuin.yml」が見えていればOK
6. 下の「Commit changes」ボタンを押す

## パート4:鍵とフォルダURLを登録する(約5分)

1. リポジトリの上のタブから「**Settings**」を開く
2. 左メニューの「Secrets and variables」→「**Actions**」
3. 「New repository secret」を押す
   - Name: `GDRIVE_SERVICE_ACCOUNT`
   - Secret: パート1-7でダウンロードした**JSONファイルをメモ帳で開き、
     中身を全部コピーして貼り付け**
   - 「Add secret」
4. すぐ隣の「**Variables**」タブに切り替え →「New repository variable」
   - Name: `SOURCE_FOLDER` / Value: 監視フォルダのURL(そのまま貼る)
   - もう1つ作成 → Name: `OUTPUT_FOLDER` / Value: 保存先フォルダのURL

## パート5:動作確認(約5分)

1. リポジトリ上部の「**Actions**」タブを開く
   (「ワークフローを有効にしますか?」と出たら有効にする)
2. 左の「刻印切り出し」をクリック →右側の「Run workflow」→ 緑のボタン
3. 少し待つと実行が一覧に出る。クリックすると進行状況が見られる
4. テスト用の動画を監視フォルダに入れてから実行すると、
   数分後に保存先フォルダに kokuin_0001.jpg ができていれば成功!

以後は**30分おきに自動実行**されます。何もしなくてOKです。

## よくある質問

**Q. お金はかかる?**
かかりません。非公開リポジトリの無料枠は月2,000分で、この仕組みは
新着がない時は1分程度で終わるため、30分おき実行+動画数本/日でも
無料枠に収まる設計です。もし超えそうなら実行間隔を1時間に広げます
(.github/workflows/kokuin.yml の cron の 30 を 0 に変える)。

**Q. 動画を上げてから画像ができるまでどれくらい?**
最大30分+処理時間(動画1本あたり数分)です。

**Q. 失敗したらどうやって気づく?**
GitHubから「Workflow failed」というメールが届きます。
Actionsタブで赤い×の実行を開くと日本語のログが見られるので、
その内容を相談してください。

**Q. 鍵ファイル(JSON)は安全?**
GitHubのSecretsは暗号化保管され、画面にも二度と表示されません。
ただし鍵ファイル自体はDriveへの入口なので、メールで人に送ったり
共有フォルダに置いたりしないでください。
