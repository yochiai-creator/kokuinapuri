# 刻印切り出し 自動処理

Google Driveの監視フォルダに動画をアップロードすると、**30分以内に自動で**:

1. 刻印(文字・数字)が最も正面で鮮明に読めるフレームを選定
2. 刻印部分を切り出して `kokuin_0001.jpg` のような連番画像として保存先フォルダへ保存
3. `処理結果一覧.csv` に処理日時・動画名・画像名・読み取れた文字を記録

## 日常の使い方

**監視フォルダに動画を入れるだけ。** 他の操作は不要です。

- すぐ処理したいとき → 上の **Actions** タブ → 「刻印切り出し」→ **Run workflow**
- 処理の様子を見たいとき → **Actions** タブで実行をクリックするとログが見られます
- 失敗したとき → GitHubからメールが届きます。ログの内容を確認してください

## 設定(Settings → Secrets and variables → Actions)

| 種類 | 名前 | 内容 |
|---|---|---|
| Secret | `GDRIVE_SERVICE_ACCOUNT` | サービスアカウントの鍵JSON(中身まるごと) |
| Variable | `SOURCE_FOLDER` | 監視フォルダのURL |
| Variable | `OUTPUT_FOLDER` | 保存先フォルダのURL |

※両フォルダにサービスアカウントのメールアドレスを「編集者」で共有しておくこと。

## 調整(必要なときだけ)

`.github/workflows/kokuin.yml` の env に以下を追加すると挙動を変えられます:

- `FRAME_INTERVAL_SEC` … フレームを調べる間隔(既定 1 秒)
- `CROP_MARGIN` … 切り出し時の余白(既定 0.2)
- `MIN_CONFIDENCE` … 文字とみなす信頼度の下限(既定 0.3)
- `OCR_LANGS` … 刻印に日本語が含まれるなら `ja,en`

実行間隔を変えるには `cron: "*/30 * * * *"` の 30 を変更(例: `"0 * * * *"` で1時間おき)。

## 注意

- このリポジトリは **Private のまま**にしてください
- Streamlit等へのデプロイは不要です(GitHub上で自動実行される仕組みです)
- `processed.json` は処理済み動画の記録です。削除すると全動画が再処理されます
